    Command

    Extends:

    utilities~ValueTracker → Command

    Direct Subclass:

    Command_delete, Command_edit, Command_link, Command_load, Command_new, Command_unlink

    The main class of the command design-pattern used in this project. There are commands for creating new entities, loading them from an asynchronous source, editing them and deleting them; and for linking or unlinking entities through specific relationships. A command can be run, rolled back and committed asynchronously. Commands have prerequisite commands. For example, a command for editing a lyph (e.g., changing its name) has to be preceded by a command for creating or loading that lyph.
